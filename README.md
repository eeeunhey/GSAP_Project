

# 💚 Learn GSAP 


**GSAP(GreenSock Animation Platform)**은
시간의 흐름에 따라 화면의 요소(위치, 크기, 색상, 투명도 등)를 **정확하고 부드럽게 변화시키는 자바스크립트 애니메이션 엔진**입니다.

이 저장소는
👉 “왜 GSAP을 써야 하는지”,
👉 “무엇을 배워야 하는지”,
👉 “어디에 적용할 수 있는지”
를 **작게 쪼개고 눈으로 직관적으로 이해할 수 있게** 정리했습니다.

---

## 1) 왜 GSAP을 써야 할까?

### ⚠️ 문제 A: CSS 애니메이션의 한계

* 단순한 움직임(fade, slide)은 가능하지만
  여러 요소가 **순서대로 등장**하거나,
  서로 **시간 차를 두고 동작**할 땐 구조가 복잡해지는 문제가 발생합니다.
* 타이밍, easing(가속도), 반복 제어 등 정밀한 조작이 어렵습니다.

### ⚙️ 문제 B: 여러 애니메이션을 한 흐름으로 관리하기 어려움

* “버튼 → 텍스트 → 이미지” 순서로 자연스럽게 이어지게 만들려면
  CSS로는 여러 keyframe을 엮기 어렵고, 여러군대 수정해야 한다.

### 🎨 문제 C: CSS로 다룰 수 없는 영역이 많음

* **SVG**, **Canvas**, **3D 객체(WebGL)**,
  혹은 수치 기반 데이터 시각화 같은 영역은 CSS로 제어 불가능합니다.

---

### 💡 GSAP를 사용함으로서 해결할 수 있는 것 


* `to()`, `from()`, `fromTo()`로 **속성 변화 애니메이션** 구현
* `timeline()`으로 **여러 움직임을 한 흐름으로 연결**
* `ScrollTrigger`로 **스크롤 위치와 애니메이션을 연동**
* 모든 수치를 **정밀하게 제어**할 수 있고,
* SVG, Canvas, WebGL 등 **모든 시각 요소에 적용 가능**

## 2) 배우는 이유와 목표 (학습 목표)

GSAP을 배우면 “움직임”을 단순히 꾸미는 것이 아니라
**시간·속성·가속도**를 논리적으로 다루는 능력을 얻게 됩니다.

| 학습 개념                   | 설명                                                      |
| ----------------------- | ------------------------------------------------------- |
| **Tween**               | ‘어떤 값’을 ‘얼마의 시간 동안’ 바꿀지 정의하는 기본 단위 (예: x좌표를 0→200으로 이동) |
| **Timeline**            | 여러 애니메이션을 순서대로 실행하거나 겹쳐서 조합하는 구조                        |
| **Ease**                | 움직임의 속도 변화를 결정하는 가속도 곡선 (느리게 시작, 빠르게 끝나는 등)             |
| **ScrollTrigger**       | 스크롤 위치를 시간처럼 활용하여 애니메이션 실행 시점 제어                        |
| **State-driven Motion** | 데이터나 상태 변화에 따라 애니메이션이 반응하도록 설계                          |
| **SVG/Canvas Motion**   | 도형이나 경로(path) 등의 수치를 직접 조작해 시각적 표현 강화                   |

---

## 3) 배움을 적용하기\

| 상황               | 예시                         | 설명              |
| ---------------- | -------------------------- | --------------- |
| **온보딩 / 히어로 섹션** | 텍스트가 한 글자씩 등장, 배경이 천천히 밝아짐 | 첫인상 강화, 몰입감 높이기 |
| **스크롤 스토리텔링**    | 스크롤 시 이미지나 텍스트가 순차적으로 나타남  | 사용자 시선 유도       |
| **버튼 / 카드 인터랙션** | hover 시 살짝 커지거나 눌리는 반응     | 직관적 피드백 제공      |
| **데이터 시각화**      | 점수·그래프가 자연스럽게 증가           | 정보를 감각적으로 전달    |
| **SVG 애니메이션**    | 로고가 ‘그려지는 듯한’ 효과           | 브랜드 완성도 향상      |

## 4) 학습 단위 (작게, 단계적으로 배우기)

### 📘 01. Tween Basics — “움직임의 최소 단위 이해하기”

> 목표: `to()`, `from()`, `fromTo()`의 차이를 이해하고, 하나의 요소를 자유롭게 움직이기

* **핵심 포인트**

  * `to()` : 현재 상태 → 지정된 값으로 이동
  * `from()` : 지정된 시작값 → 현재 상태로 이동
  * `fromTo()` : 시작값과 종료값 모두 직접 지정
* **배울 내용**

  * `duration`, `delay`, `repeat`, `yoyo`
  * position, scale, opacity 등의 기본 속성
* **예시**

  ```js
  gsap.to(".box", { x: 200, duration: 1, ease: "power2.out" });
  ```

---

### 📗 02. Ease Lab — “움직임의 느낌 조절하기”

> 목표: 같은 거리라도 **속도 곡선**(Ease)에 따라 다르게 느껴짐을 체험하기

* **Ease란?**
  단순히 빠르게 움직이는 것이 아니라,
  *“어디서 빨라지고 어디서 느려지는가”*를 제어하는 함수
* **대표 예시**

  * `power1.inOut`: 부드럽게 시작, 부드럽게 끝
  * `back.out(1.7)`: 튀어나오듯 반동
  * `bounce.out`: 공처럼 튀는 느낌
* **배울 내용**

  * `ease` 속성 실험
  * CustomEase(직접 커브 그리기)

---

### 📙 03. Timeline Fundamentals — “여러 애니메이션을 연결하기”

> 목표: 애니메이션의 순서, 겹침, 지연을 제어하는 타임라인 구조 익히기

* **핵심 포인트**

  * `gsap.timeline()`으로 여러 애니메이션을 한 흐름으로 연결
  * `" -=0.2 "` 같은 **position parameter**로 겹침 조정
* **배울 내용**

  * `add()`, `from()`, `to()`의 조합
  * `tl.play()`, `tl.pause()`, `tl.reverse()`
* **결과**
  자연스럽게 이어지는 “하나의 장면” 완성

---

### 📒 04. ScrollTrigger Core — “스크롤로 움직임을 제어하기”

> 목표: 스크롤 위치를 애니메이션의 ‘시간축’처럼 다루기

* **핵심 포인트**

  * `start`, `end`, `scrub`, `pin`
  * “페이지를 내릴 때 → 요소가 움직이는 시점” 제어
* **배울 내용**

  * 스크롤에 따라 텍스트 등장
  * 섹션 고정(Pinning)
  * 구간별 강조 효과
* **예시**

  ```js
  gsap.to(".section", {
    y: -100,
    scrollTrigger: { trigger: ".section", start: "top 80%", scrub: true },
  });
  ```

---

### 📕 05. SVG Motion — “그림처럼 움직이는 애니메이션”

> 목표: SVG의 path, stroke 등을 제어해 **‘그려지는’ 모션** 만들기

* **핵심 포인트**

  * `stroke-dasharray`, `stroke-dashoffset` 개념
  * 로고나 선이 ‘그려지는’ 느낌 구현
* **배울 내용**

  * path 길이 계산, 드로잉 효과
  * 색상·투명도 변화와 결합

---

### 📔 06. State-driven Motion — “데이터에 반응하는 애니메이션”

> 목표: 감정 점수, 사용자 입력 등 **변하는 값에 따라 움직임** 연동하기

* **핵심 포인트**

  * 실시간 값 변화 → `gsap.to()`로 자연스럽게 전이
  * 예: “감정 점수에 따라 바의 길이 변화”
* **예시**

  ```js
  gsap.to(".emotion-bar", { width: score * 100 + "%", duration: 0.6 });
  ```

---

### 📓 07. Performance Notes — “부드럽고 효율적인 모션 만들기”

> 목표: 성능 저하 없이 애니메이션이 매끄럽게 동작하도록 이해하기

* **핵심 포인트**

  * transform(x, y, scale, rotate) 우선 사용
  * top/left 변경은 레이아웃 리플로우 발생 → 피하기
  * 너무 많은 요소 동시 애니메이션 → GPU 과부하 주의
* **배울 내용**

  * layout thrash 개념
  * `will-change` 활용법
  * requestAnimationFrame vs GSAP 성능 비교

---

### 📘 08. Pattern Gallery — “자주 쓰는 패턴 정리”

> 목표: 실무에서 자주 마주치는 애니메이션 유형들을 GSAP으로 구현하기

* **포함 패턴**

  * Hero Section Fade/Slide
  * Button Hover 반응
  * Chart Transition (값 변화 시 애니메이션)
  * Modal / Popup 등장/퇴장
  * Scroll-based Card Animation

## 6) 빠른 비교 (언제 GSAP이 좋은가)

| 상황              | 적합한 방법             | 이유              |
| --------------- | ------------------ | --------------- |
| 단순 hover, fade  | CSS animation      | 간단하고 가벼움        |
| 여러 요소 순서 제어     | GSAP timeline      | 구조적, 유지보수 용이    |
| 스크롤 연동          | GSAP ScrollTrigger | 구간별 정밀 제어 가능    |
| SVG / Canvas 제어 | GSAP core          | CSS로 불가능한 영역 가능 |
